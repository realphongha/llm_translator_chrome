# LLM Page Translator
*A Chrome extension for translating webpages using any OpenAI-compatible API.*

---

# Goals

- Lightweight and fast.
- Works with any OpenAI-compatible endpoint.
- Supports local models (llama.cpp, Ollama, LM Studio, etc.).
- Supports cloud providers (OpenRouter, OpenAI, DeepSeek, etc.).
- Paragraph-aware translation.
- Optimized for web novels (Qidian) while remaining general-purpose.
- No account system.
- No backend server.
- Everything stored locally.
- Make everything event-driven.

---


```
You are a professional translator.

Translate from {{source_language}} to {{target_language}}.

Rules:

- Preserve paragraph structure.
- Do not summarize.
- Do not explain.
- Output only the translation.
```

```
- Keep dialogue natural.
```

# User Prompt

The user prompt is shared by every system prompt.

Example:

```

Each paragraph begins with an ID.

Return one translated paragraph for each ID.

<ID=1>

...

<ID=2>

...

<ID=3>
```

This keeps batching logic independent of translation style.

# Modules

## 1. DOM Observer

Responsibilities:

- detect new content
- detect page navigation (SPA)
- observe infinite scrolling
- enqueue new blocks

Default:

Observe entire document.

Uses:

```
MutationObserver
```

```

.ads
```

## 3. Text Extractor

Responsible for:

- extracting translatable nodes
- ignoring:

```
script

style

textarea

input

svg

code

pre
```

Skip empty nodes.

Skip whitespace.

Skip already translated nodes.

## 4. Translation Queue

Every paragraph becomes:

```ts
{
    id,
    text,
    element
}
```

Pipeline:

```
DOM

↓

Queue

↓

Batch Builder

↓

API Queue

↓

Renderer
```

---

# Batch Builder

Collect paragraphs until:

```
max_chars
```

or

```
queue empty
```

Example:

```
P1

P2

P3

P4
```

↓

One API request.

---

If one paragraph exceeds limit:

```
split

↓

Part A

Part B

Part C
```

Merge before rendering.
