# LLM Page Translator v0.1.0

*A Chrome extension for translating webpages using any OpenAI-compatible API.*

---

## Goals

- Lightweight and fast
- Works with any OpenAI-compatible endpoint
- Supports local models (llama.cpp, Ollama, LM Studio, etc.)
- Supports cloud providers (OpenRouter, OpenAI, DeepSeek, etc.)
- Paragraph-aware translation
- Optimized for web novels (Qidian) while remaining general-purpose
- No account system
- No backend server
- Everything stored locally
- Event-driven architecture

---

## Features Implemented

### Core Pipeline

```
DOM
  │
  ├─ TreeWalker text extraction
  ├─ Priority computation via CSS rules
  │
  ▼
Enqueue → Background via chrome.runtime.sendMessage
  │
  ├─ Priority sort (lower number = higher priority)
  ├─ Parallel workers (configurable, default 32)
  ├─ Cache check (SHA-256 key, true LRU eviction)
  ├─ Batch builder (group by maxChars, split oversized)
  │
  ▼
API Call (POST /v1/chat/completions)
  │
  ├─ Retry with exponential backoff (1s, 2s, 4s)
  ├─ Parse <ID=N> tagged response
  ├─ Merge split paragraphs
  │
  ▼
Content Script
  │
  ├─ Apply translations to DOM
  ├─ State indicators (⟳ translating / ✓ translated / ⚠ error)
  └─ Stuck element recovery (2s after queue empties)
```

### Translation Cache

```
Key:
  SHA-256(systemPrompt + "\x00" + model + "\x00" + text)

Value:
  { translation: string, timestamp: number }

Eviction:
  - True LRU (reads update recency order in index)
  - Hard count cap: 10,000 entries
  - Byte-based cap: configurable (default 7 MB, via chrome.storage.local.getBytesInUse)
  - Oldest entries evicted first (front of index)

Storage:
  chrome.storage.local with "cache_" prefix
```

### Per-Site Settings

Stored in `chrome.storage.local` under key `"site_<hostname>"`. Auto-created on first visit.

| Field | Type | Default | Notes |
|---|---|---|---|
| Mode | `"off" \| "on" \| "auto"` | `"off"` | `off` = do nothing; `on` = manual (floating bar, no auto on load); `auto` = auto-translate on load |
| Prompt | `string` | `"general"` | ID of built-in or user prompt |
| Source Language | `string` | `"Auto"` | Source language name |
| Target Language | `string` | `"English"` | Target language name |
| Observe | `string` | `"body"` | CSS selector for MutationObserver root |
| Content Selector | `string` | `""` | CSS selector for paragraphs (empty = auto) |
| Ignore | `string[]` | `[]` | CSS selectors for ignored elements |
| Priority Rules | `PriorityRule[]` | `[]` | CSS selector → priority number |

> **Legacy migration:** a stored boolean `enabled` is migrated to `mode` (`enabled=true` → `auto`, `enabled=false` → `off`) on load. New installs use `mode` directly.

### DOM Observer

- `MutationObserver` with 400ms debounce
- Observes `childList: true, subtree: true` on configured root (default `document`)
- SPA navigation: `popstate`, `hashchange`, patched `history.pushState`/`replaceState`
- 800ms delay after URL change before re-scanning entire body
- 300ms initial delay for client-side framework render

### Text Extractor

- Walks text nodes via `TreeWalker` inside matching scope elements
- Skips: `<script>`, `<style>`, `<textarea>`, `<input>`, `<select>`, `<button>`, `<svg>`, `<canvas>`, `<code>`, `<pre>`, `<kbd>`, `<samp>`, `<math>`, `<head>`, `<link>`, `<meta>`, `<title>`
- Skips elements matching ignore selectors (checks ancestors via `el.closest()`)
- Skips already-translated elements (`[data-translation-id]`)
- **Re-attaches re-rendered translated text via in-session translation memory**: if a text node already matches a known `translated → original` mapping (e.g. after a Vue/React re-render dropped the wrapper span), it is re-wrapped with the correct `data-original` and not re-sent to the model
- Strips trailing spinner character (` ⟳`)
- Skips empty text nodes and non-letter text
- Wraps each text node in `<span data-translation-id="N" data-original="..." data-translation-state="waiting">`
- Computes priority via `el.closest()` against priority rules (lowest matching wins)

### Priority System

- Lower number = higher priority (1 = highest, default = `Infinity`)
- Items sorted by priority, then by enqueue order (FIFO)
- Priority rules defined per-site: `{ selector: string, priority: number }`
- Used in both extraction (sets `data-priority`) and queue (sorting)

### State Indicators (CSS)

- `"translating"` → spinning ⟳ icon, blue color, 0.7 opacity
- `"translated"` → clean (no indicator), full opacity
- `"error"` → ⚠ icon, red color, wavy underline
- `"waiting"` → no indicator

### DOM Scan (Popup Debug Feature)

Groups DOM elements by CSS path (4 levels deep, stops at `#id`). Heuristic:

| Count | Suggestion | Priority |
|---|---|---|
| ≥15 | "content" (likely main content) | 1 |
| 5-14 | "priority" (moderate frequency) | 50 |
| <5 | "ignore" (likely nav/UI) | 99 |

One-click buttons: "Use as Content" → sets content selector, "Ignore" → adds to ignore list, "Priority N" → adds priority rule.

### Key Design Decisions

1. **No streaming API** — Simple request/response reduces complexity. Results are still progressive via per-sub-batch emission.

2. **IIFE bundles** — Chrome extension CSP restrictions make IIFE the safe choice over ESM for content scripts.

3. **Priority-based queue** — Items sorted on every enqueue. Processing is parallel but highest-priority items always start first (sorted to front of queue).

4. **True LRU cache** — Every cache hit updates recency order (key moved to end of index). Dual eviction: hard count cap (10K) + byte-based cap (default 7 MB).

5. **Retry logic** — 3 retries with exponential backoff (1s, 2s, 4s). HTTP 401 is not retried (auth error).

6. **Stuck element recovery** — Elements stuck in "translating" state are reverted and re-enqueued 2s after queue empties.

7. **Config auto-reload** — Content script listens to `chrome.storage.onChanged` and re-initializes automatically.

8. **Three-mode model (`off` / `on` / `auto`)** — Replaces the old boolean enable toggle. `off` does nothing; `on` shows the floating bar and stays strictly manual until the user clicks Translate (which then starts the observer for lazily-loaded content); `auto` translates on load and observes. Mode changes trigger an immediate retranslate so they take effect without a reload.

9. **In-session translation memory** — Maps `translated text → original text`, populated when a translation is applied. Lets framework re-renders be recognized and re-attached with the correct `data-original` instead of being mis-captured as new source (the root cause of the "data-original holds translated text" corruption on SPA sites). General across sites (keyed by translated text, not a site-specific index). Cleared on Retranslate/reload.

10. **Retranslate bypasses cache** — The floating-bar and per-paragraph Retranslate use `skipCache: true` so the model always produces a fresh translation rather than re-applying the cached one.

11. **Manual translation** — Per-paragraph "Translate Manually" writes the user-supplied text into the translation cache under the same key the model would use, so it persists and behaves like a normal LLM result.

12. **Reversible Show Original toggle** — Page-wide swap between source and translated text is non-destructive: current text is stashed in `data-translated` when showing the original and restored on toggle-back, with no API calls.

13. **Disable Google Translate on active sites** — For `on`/`auto` sites the content script injects `<meta name="google" content="notranslate">` into `document.head` so Chrome's built-in translator and Google's translation service don't conflict with our own translation. Injected at `document_idle` (after mode is known), idempotent. `off` sites are left untouched.
